I built a free, open source tool called **SpeciesNet Studio** that gives you a web UI for reviewing Google SpeciesNet predictions on your own images.

**The short version:** point it at your image folder, run SpeciesNet from inside the app, then review the results in a gallery — approve, override, or flag each image with a single keypress. Export to CSV when done. Everything runs on your computer, nothing goes to a server.

**It's free, no subscription, no sign-up.** Download and run with Docker: https://github.com/arunrajiah/speciesnet-studio

If you don't want to load your own images to try it, there's a "Try with sample data" button that loads some Creative Commons wildlife images so you can see the workflow immediately.

**One question for this group:** what cameras and folder structures do you use? I want to make sure the image ingestion handles the way SD cards actually come off cameras in the field — especially any naming conventions or nested folder layouts that might be unusual. If something doesn't import correctly, let me know and I'll fix it.

GitHub issues: https://github.com/arunrajiah/speciesnet-studio/issues
